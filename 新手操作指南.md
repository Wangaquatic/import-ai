# 🚀 项目运行指南 - 给组员的完整操作步骤

## 📋 前置准备

在开始之前，请确保你的电脑上安装了以下软件：

### 1. 安装 Node.js

**Node.js 是什么？** 它是运行 JavaScript 代码的环境，我们的项目需要它来安装依赖和运行。

**下载地址：** https://nodejs.org/

- 推荐下载 **LTS（长期支持版）**，目前是 18.x 或更高版本
- 下载后双击安装包，一路点击"下一步"即可
- 安装完成后，打开命令行（Windows 按 `Win + R`，输入 `cmd`），输入以下命令验证：

```bash
node -v
npm -v
```

如果显示版本号（如 `v18.17.0` 和 `9.6.7`），说明安装成功！

### 2. 安装 Git（可选，但推荐）

**Git 是什么？** 版本控制工具，用于下载和管理代码。

**下载地址：** https://git-scm.com/

- 下载后安装，保持默认选项即可
- 验证安装：

```bash
git --version
```

### 3. 安装代码编辑器（推荐 VSCode）

**VSCode 是什么？** 一个强大的代码编辑器，方便查看和修改代码。

**下载地址：** https://code.visualstudio.com/

- 下载后安装即可

## 📦 获取项目代码

### 方式一：使用 Git 克隆（推荐）

如果你安装了 Git，打开命令行，进入你想存放项目的文件夹，然后运行：

```bash
git clone <项目的Git地址>
cd import-ai
```

> **注意：** 将 `<项目的Git地址>` 替换为实际的仓库地址（如 GitHub、GitLab 链接）

### 方式二：直接下载压缩包

如果没有 Git，可以：

1. 从项目分享链接下载 ZIP 压缩包
2. 解压到你想要的文件夹
3. 打开命令行，使用 `cd` 命令进入项目文件夹：

```bash
cd 你的文件夹路径/import-ai
```

---

## 🔧 安装项目依赖

进入项目文件夹后，运行以下命令安装所有需要的依赖包：

```bash
npm install
```

**这一步会做什么？**
- 自动下载项目需要的所有第三方库（如 React、Electron 等）
- 可能需要 3-10 分钟，取决于网络速度
- 完成后会在项目中生成一个 `node_modules` 文件夹

**如果遇到网络问题：**

可以尝试使用国内镜像源：

```bash
npm config set registry https://registry.npmmirror.com
npm install
```

---

## 🎮 运行项目

### 开发模式（推荐用于测试和开发）

```bash
npm run dev
```

**会发生什么？**
- 自动编译代码
- 打开一个 Electron 桌面应用窗口
- 支持热更新（修改代码后自动刷新）

**如何停止？**
- 在命令行窗口按 `Ctrl + C`

### 预览模式

```bash
npm start
```

这是另一种运行方式，用于预览构建后的效果。

## 📦 打包成可执行文件

如果你想生成一个可以直接双击运行的应用程序（不需要命令行），可以使用以下命令：

### Windows 系统

```bash
npm run build:win
```

打包完成后，在 `dist` 文件夹中会生成 `.exe` 安装包。

### macOS 系统

```bash
npm run build:mac
```

打包完成后，在 `dist` 文件夹中会生成 `.dmg` 安装包。

### Linux 系统

```bash
npm run build:linux
```

## 🛠️ 常见问题解决

### 问题 1：`npm install` 失败

**可能原因：**
- 网络问题
- Node.js 版本过低

**解决方法：**
1. 检查 Node.js 版本是否 >= 18
2. 切换到国内镜像源（见上文）
3. 删除 `node_modules` 和 `package-lock.json`，重新运行 `npm install`

### 问题 2：`npm run dev` 报错

**可能原因：**
- 依赖没有安装完整
- 端口被占用

**解决方法：**
1. 确保 `npm install` 成功完成
2. 重启电脑后再试
3. 查看错误信息，搜索具体错误提示

### 问题 3：打包失败

**可能原因：**
- 磁盘空间不足
- 权限问题

**解决方法：**
1. 确保有足够的磁盘空间（至少 2GB）
2. 以管理员身份运行命令行
3. 检查 `electron-builder.yml` 配置是否正确

## 💡 开发建议

